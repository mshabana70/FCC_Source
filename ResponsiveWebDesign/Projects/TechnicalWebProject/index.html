CSS Flexbox

  • The Flexibile Box Module (also know as 'flexbox'), is a one dimensional layout model. This module is used as a method for space distribution between webpage elements in a user interface, and also has powerful item alignment capabilities.

    When we say 'One Dimensional,' what we mean by this is that Flexbox handles layouts in one dimension at a time. With flexbox, you are creating a layout either as a row or as a column, never both at the same time. This is different from CSS Grid Layout, which is capable of styling layouts for columns and rows similtaneously.

Flexbox Axes

  • There are two axes that a developer should consider when working with flexbox - the main axis and the cross axis.

    The Main Axis

    The main axes is defined by the flex-direction, which has four possible values: row, row-reverse, column, column-reverse.

    With the row or row-reverse, the main axis will run along the inline or horizontal direction.

  • The Cross Axis

    The cross axis is perpendicular to the main axis, defined by the flex-direction. If the main axis is set to row or row-reverse, then the cross axis would be in the direction of the columns.

    If the main axis is set to column or column-reverse, then the cross axis would be in the direction of the rows.

    It is important to understand the axis you are working on. It will become useful when we begin to discuss aligning and justifying the items within our flex-container.

Flexbox Lines

  • When working with flexbox, we no longer make assumptions about the writing standards of a document. Originally, most documents on the web would start from the left side of the page and carry text over to the right side of the page. Modern layouts have no assumptions and support a wide range of writing modes, whether that be from left-to-right or right-to-left.

Flex Container

  • An area or div of a document that is designated for using flexbox is called a flex container. To create a flex container, we set the value of the container's display property to flex or inline-flex. Once the value is changed, the children of the flex container become flex items. Since most properties have initial values defined, it is important to understand what those initial values are when defining a flex container.

    • Flex items will display in a row (flex-direction has a value of row).
    • The flex items begin from the start edge of the main axis (left to right).
    • The flex items dont stretch, but can shrink.
    • The items will stretch to fill size of cross axis.
    • The flex-basis property has value of auto.
    • The flex-wrap property has value of nowrap.

Multi-Line Container

  • Flexbox has an intuitive method to creating multi-line containers. Although flexbox is a one dimensional box model, it is possible for our flex items to 'wrap' onto multiple lines. Once those flex items are wrapped to another line, it is best to consider each of those lines are their own flex container. All distribution rules that were written for the initial flex container will occur across those lines, without reference to the lines either side.

    To implement this 'wrap' behavior, use the flex-wrap property with a value of wrap. With this property, all flex items that are too large to display on one line, will wrap onto another line below.

Properties for Flex Items

  • There are three properties that allow developers to target flex items directly. To have control of flex items we use:

    • flex-grow
    • flex-shrink
    • flex-basis

    What is actually controlled by these flex item properties is the available space within our flex container. In the example below, we have three flex items that are 100 pixels wide in a flex container that is 500 pixels wide. With the flex items flush together, that leaves an available space of 200 pixels wide. By default, the flex container will push that available space to the right of the last flex item.

  • The flex-basis property

    The flex-basis property defines the size of the flex item it is applied to. The initial value of this property is auto, which the browser looks to see if the items have a size. Based on the previous example, all of the flex items above have a width of 100 pixels, which would be used as the value for flex-basis.

    If no size is specified for the flex items, then the content's size is used as the flex-basis. For more on the flex-basis property, visit MDN Web Docs.

  • The flex-grow property

    flex-grow property defines the stretch of the flex items within your flex container. Setting the value to a positive integer will allow the flex items to grow along the main axis from their flex-basis.

    Say we applied the flex-grow property to all the examples above with a value of 1. The available space in the flex container would be equally shared between the flex items, and they would stretch to fill the container on the main axis.

    If we were change flex-grow property for the first flex item to 2, this would change the ratio of available space given to each flex item. Out of the 200 pixels of available space, the first flex item would take 100 pixels compared to the second and third item's 50 pixels. Resulting in the available space to be distributed on a ratio of 2:1.

  • The flex-shrink property

    Works similarily to the flex-grow property, but rather than give available space to flex items, flex-shrink takes away available space. When applying a positive value to the flex-shrink property, the flex item can now shrink smaller than its flex-basis. An item with a larger positive value for flex-shrink will shrink faster than other flex items that have lower values. For more information on the details of these properties, visit MDN Web Docs.
